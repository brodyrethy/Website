Autocomplete

This is a vim function that I never see people using. Instead, most opt to use plugins that encorporate an different autocomplete function to fill the gap (a good one being supertab). Little do they know, it already exists by default...

While in Insert Mode
Ctrl+N/P
Ctrl+X Ctrl+N/P

Both keybindings achieve the same thing. Ctrl+N/P selects the next or previous item in the autocomplete list.

A Built in (Fuzzy) File Finder

By simply adding

set path+=**

to your .vimrc, you enable the option to search recursively using vim's :find command.

Rebind Colon (Command Mode) to Semi-Colon

:nmap ; :

Quite simple and effortless compared to original Shift+Semi-Colon key chord.

Infinite File Modification History

set undofile and set undodir=~/.vim/undodir. This will make vim save all change history for every file. You can find the change files in ~/.vim/undodir.

Relative Line Number

Relative line numbers allow you to see the lines relative to you both positively and negatively. This allows you to easily spot the exact line you want to go to using line number + Return or - line number + Return.

No Need For a Nerdtree; Use Netrw and the Leader Key

Note: The Leader key is typically mapped to \ (forward slash).

Instead of using a nerdtree to move through directories/files, use netrw and the leader key. I typically take the most common directories I spend time in, and bind their paths to leader like this:

:map ghh :e ~/1TBDrive/github

As shown by the code, I change to the github directory by hitting

\ g h h

in quick succession. This enables me to quickly move around to various working directories. I also take the :Explore command, which opens your current working directory in netrw.

Backspace in Insert Mode

I think most vimmers already know about this, but if you don't, you'll probably want to enable it, as I feel it's essential:

set backspace=indent,eol,start

This will enable you to use backspace while in Insert mode.
