Why Window Managers Produce the Fastest Workflow

What is a Window Manager?

First, a definition of a window manager. A window manager is simply a piece of software which controls the placement of individual windows.


What is a Floating Window Manager?

In a floating window manager, each window floats. Typically, a floating window manager is the default type you will see on Mac OS X, Windows, and Linux (GNOME, XFCE, Openbox, etc.).


What is a Tiling Window Manager?

A tiling window manager is similar to the last definition but that it functions in a grid style instead of a floating one, where everything is displayed on the screen at once. More so, each window is either a master, or a slave. First, you open the master. When another window is opened, the master is divided where the new window is the master and the last is the slave. Then, when another is opened, the newest is the master, and the last two are slaves, and so on, so forth...



Image taken from https://i3wm.org/screenshots/

i3wm: a manual tiling window manager (you choose how new windows are split, be it vertical or horizontally, and windows can be re-sized).



Image taken from https://dwm.suckless.org/screenshots/

dwm: a dynamic tiling window manager (Windows are split automatically; Master window on the left, slave windows on the left in "default" layout).


The Speed Differences

Speed differences were obtained by comparing the number of key presses to accomplish various actions using each window manager. To simplify the testing process, I chose to compare just Windows 10 versus dwm and i3wm. i3wm and dwm were chosen because they are both two different sub-types of tiling window managers, being manual and dynamic.

NOTE: I chose to count a keyboard "chord" not as one entity, but instead the sum of keys being pressed. I made note of when a keyboard chord is used.
